# Security policy & posture score

_The org security policy and the live posture score derived from it._

Settings → Security is the org-wide security policy: authentication (MFA, SSO, methods, step-up), sessions, network & devices, API access, and data protection. A live posture score (0–100, RAG-rated) reacts as you change controls.

## Configure → observe

The same posture score and its prioritized gaps appear on Admin Security and Fleet Posture, so what you set in the policy is what you see across the observability surfaces. The score is weighted — MFA, phishing-resistant factors, enforced SSO, and an enforced IP allowlist move it most.
