# Rich content: media, callouts & layout

_Every formatting block you can drop into a document, with a live example of each._

All of these work in **any** `.mdx` file — published or internal — with **no import line**. Just type the tag. The same set powers the visual editor, so what you write here is exactly what point-and-click will insert later.

<Callout type="tip" title="Starting a new document?">
  Run `npm run new-doc` (see [the authoring overview](/how-to)) to scaffold a
  file with the right frontmatter in the right folder — then paste any of the
  blocks below.
</Callout>

## Callouts

Colored, titled admonition boxes. Types: `info`, `tip`, `success`, `warning`, `danger`, `note`.

<Callout type="info" title="Note">This is an informational callout.</Callout>
<Callout type="warning" title="Rotate keys regularly">API keys should be rotated every 90 days.</Callout>
<Callout type="danger" title="Irreversible">Deleting a policy pack cannot be undone.</Callout>

```mdx
<Callout type="warning" title="Rotate keys regularly">
  API keys should be rotated every 90 days.
</Callout>
```

## Video — YouTube

Paste a full YouTube link **or** just the id; both work.

<YouTube url="https://www.youtube.com/watch?v=aqz-KE-bpKQ" />

```mdx
<YouTube url="https://www.youtube.com/watch?v=aqz-KE-bpKQ" />
<YouTube id="aqz-KE-bpKQ" start={30} />
```

## Video — self-hosted file

For an `.mp4`/`.webm` you host yourself. Put the file under `public/media/` and reference it as `/media/<file>`.

```mdx
<Video src="/media/product-tour.mp4" poster="/media/product-tour.jpg" caption="A 90-second tour." />
```

## Images with captions

Use `<Figure>` when you need a caption, sizing, or alignment. Plain Markdown `![alt](/path)` still works for a bare image.

<Figure src="/diagrams/architecture.svg" alt="PLCY architecture" caption="The control-plane / data-plane split." border />

```mdx
<Figure src="/diagrams/architecture.svg" alt="PLCY architecture"
        caption="The control-plane / data-plane split." border />
```

## Links & buttons

A call-to-action link styled as a button. Internal paths route instantly; external URLs open in a new tab.

<Button href="/technical/api-keys">Read the API guide</Button>
<Button href="https://plcy.app" variant="secondary">Visit plcy.app</Button>

```mdx
<Button href="/technical/api-keys">Read the API guide</Button>
<Button href="https://plcy.app" variant="secondary">Visit plcy.app</Button>
```

## Color & highlight

<Color value="#1f6fd0">Brand-blue text</Color>, a <Highlight>highlighted phrase</Highlight>, or a <Highlight color="#d6f5e3">green swatch</Highlight>. Custom <Font family="ui-monospace" weight={700}>font runs</Font> when you truly need them.

```mdx
<Color value="#1f6fd0">Brand-blue text</Color>
<Highlight>highlighted phrase</Highlight>
<Highlight color="#d6f5e3">green swatch</Highlight>
<Font family="ui-monospace" weight={700}>font run</Font>
```

## Badges

Inline status pills — tones: `neutral`, `blue`, `green`, `amber`, `red`.

<Badge tone="green">Available</Badge> <Badge tone="amber">Beta</Badge> <Badge tone="red">Deprecated</Badge>

```mdx
<Badge tone="green">Available</Badge> <Badge tone="amber">Beta</Badge>
```

## Columns

Side-by-side content that stacks on mobile.

<Columns count={2}>
  <Column>
    **BYOK** — bring your own model keys. You pay the provider directly.
  </Column>
  <Column>
    **PLCY-managed** — inference billed through PLCY on one invoice.
  </Column>
</Columns>

```mdx
<Columns count={2}>
  <Column>**BYOK** — bring your own model keys.</Column>
  <Column>**PLCY-managed** — one invoice.</Column>
</Columns>
```

## Card grid

A responsive grid of cards, optionally links.

<CardGrid min="14rem">
  <Card title="Architecture" href="/technical/architecture" icon="◈">
    How the control and data planes fit together.
  </Card>
  <Card title="Policy packs" href="/features/policy-packs" icon="▦">
    Controls, primitives, and composite packs.
  </Card>
  <Card title="DSAR" href="/features/incidents-dsar" icon="✎">
    Handling data-subject requests.
  </Card>
</CardGrid>

```mdx
<CardGrid min="14rem">
  <Card title="Architecture" href="/technical/architecture" icon="◈">
    How the control and data planes fit together.
  </Card>
</CardGrid>
```
